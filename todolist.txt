[ ] Rotate CRON_SECRET + WEBHOOK_SECRET into separate values before going live
    Added: 2026-04-22
    Reason: CRON_SECRET and WEBHOOK_SECRET currently share the same value
    (0cace6a1...fae4f7a) in both .env.local and Vercel. They're used for
    different purposes (CRON_SECRET authenticates Vercel Cron -> /api/cron/*;
    WEBHOOK_SECRET authenticates inbound webhooks), so a leak of one should
    not compromise the other.
    Steps:
      1. Generate two fresh tokens:
           node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
      2. Update CRON_SECRET in .env.local and in Vercel (all 3 envs).
      3. Update WEBHOOK_SECRET in .env.local and in Vercel (all 3 envs).
      4. Redeploy. Smoke-test the cron endpoint with:
           curl -H "Authorization: Bearer <new_token>" <prod_url>/api/cron/sync
      5. Smoke-test any webhook path (Twilio inbound, etc.).
